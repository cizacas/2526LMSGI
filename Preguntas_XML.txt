CUESTIONARIO XML - 20 PREGUNTAS DE OPCIÓN MÚLTIPLE
======================================================

1. ¿Qué significa XML?
   a) eXtra Markup Language
   b) eXtensible Markup Language
   c) eXecutable Markup Language
   d) eXtensive Markup Language

2. ¿Cuál es la principal diferencia entre XML y HTML?
   a) XML fue diseñado para mostrar datos, HTML para llevar datos
   b) XML fue diseñado para llevar datos, HTML para mostrar datos
   c) XML y HTML tienen el mismo propósito
   d) XML es más antiguo que HTML

3. ¿Con qué debe comenzar un documento XML?
   a) Con una etiqueta <xml>
   b) Con un comentario
   c) Con una declaración XML (opcional)
   d) Con el elemento raíz obligatoriamente

4. ¿Cuál es la declaración XML típica correcta?
   a) <?xml version="1.0" encoding="UTF-8"?>
   b) <xml version="1.0" encoding="UTF-8">
   c) <!xml version="1.0" encoding="UTF-8"!>
   d) <xml version="1.0" encoding="UTF-8"/>

5. ¿Qué indica el atributo standalone="yes" en la declaración XML?
   a) El documento tiene errores de sintaxis
   b) El documento no depende de ningún archivo externo
   c) El documento debe ser validado
   d) El documento está en modo de prueba

6. ¿Cómo se escribe correctamente un comentario en XML?
   a) // Este es un comentario
   b) /* Este es un comentario */
   c) <!-- Este es un comentario -->
   d) # Este es un comentario

7. ¿Cuáles son las reglas básicas para los nombres de elementos en XML?
   a) Pueden comenzar con números
   b) Pueden contener espacios
   c) Deben comenzar con una letra o guión bajo (_)
   d) No distinguen entre mayúsculas y minúsculas

8. ¿Cuál de estas etiquetas XML es incorrecta?
   a) <nombre>Juan</nombre>
   b) < nombre>Juan</nombre>
   c) <nombre >Juan</nombre>
   d) <nombre>Juan</nombre>

9. ¿Cuándo se recomienda usar ATRIBUTOS en lugar de ELEMENTOS?
   a) Para contenido textual largo
   b) Para identificadores únicos y metadatos simples
   c) Para datos estructurados complejos
   d) Para contenido que puede crecer

10. ¿Cuál es la sintaxis correcta para una etiqueta auto-cerrada?
    a) <imagen src="foto.jpg" alt="Foto">
    b) <imagen src="foto.jpg" alt="Foto" />
    c) <imagen src="foto.jpg" alt="Foto" \\>
    d) <imagen src="foto.jpg" alt="Foto" ></>

11. ¿Qué hace el atributo xml:space="preserve"?
    a) Elimina todos los espacios en blanco
    b) Conserva todos los espacios en blanco tal como están
    c) Reduce los espacios múltiples a un solo espacio
    d) Añade espacios automáticamente

12. ¿Cuáles son los 5 caracteres especiales principales que deben escaparse en XML?
    a) < > & ' "
    b) < > & # %
    c) ( ) [ ] { }
    d) + - * / =

13. ¿Cómo se representa el símbolo < en XML?
    a) &lt;
    b) &gt;
    c) &less;
    d) &menor;

14. ¿Qué formato tiene una referencia de carácter decimal en XML?
    a) &n;
    b) &#n;
    c) #n;
    d) &[n];

15. ¿Qué significa que un documento XML esté "bien formado"?
    a) Que tenga una buena presentación visual
    b) Que cumpla con las reglas básicas de sintaxis de XML
    c) Que tenga muchos elementos
    d) Que sea muy largo

16. ¿Cuál de estas características NO es necesaria para que un documento XML esté bien formado?
    a) Tener un elemento raíz único
    b) Las etiquetas deben estar correctamente anidadas
    c) Todos los atributos deben tener valores entre comillas
    d) Debe tener al menos 10 elementos

17. ¿Qué son los espacios de nombres en XML?
    a) Espacios en blanco entre elementos
    b) Una forma de evitar conflictos de nombres entre vocabularios XML
    c) Directorios donde se guardan archivos XML
    d) Espacios reservados en memoria

18. ¿Cómo se declara un espacio de nombres con prefijo en XML?
    a) xmlns:prefijo="URI"
    b) namespace:prefijo="URI"
    c) ns:prefijo="URI"
    d) space:prefijo="URI"

19. ¿Qué es un espacio de nombres predeterminado (default namespace)?
    a) El primer espacio de nombres declarado
    b) Un espacio de nombres que se aplica a elementos sin prefijo
    c) El espacio de nombres más usado
    d) Un espacio de nombres obligatorio

20. En este código XML: <libros xmlns:ficcion="http://ejemplo.com/ficcion">, ¿qué representa "ficcion"?
    a) El URI del espacio de nombres
    b) El prefijo del espacio de nombres
    c) El nombre del elemento
    d) Un atributo normal

=====================
1. b) eXtensible Markup Language
2. b) XML fue diseñado para llevar datos, HTML para mostrar datos
3. c) Con una declaración XML (opcional)
4. a) <?xml version="1.0" encoding="UTF-8"?>
5. b) El documento no depende de ningún archivo externo
6. c) <!-- Este es un comentario -->
7. c) Deben comenzar con una letra o guión bajo (_)
8. b) < nombre>Juan</nombre>
9. b) Para identificadores únicos y metadatos simples
10. b) <imagen src="foto.jpg" alt="Foto" />
11. b) Conserva todos los espacios en blanco tal como están
12. a) < > & ' "
13. a) &lt;
14. b) &#n;
15. b) Que cumpla con las reglas básicas de sintaxis de XML
16. d) Debe tener al menos 10 elementos
17. b) Una forma de evitar conflictos de nombres entre vocabularios XML
18. a) xmlns:prefijo="URI"
19. b) Un espacio de nombres que se aplica a elementos sin prefijo
20. b) El prefijo del espacio de nombres
